# UI/UX 全面翻新审计（2026-02-15）

## 1. 你的核心反馈
- 现状观感“像 80 年代老网站”
- UI 视觉老旧、层次不清
- UX 操作流程割裂，用户完成核心任务成本高

## 2. 本轮已落地的 UI/UX 重构

### 2.1 视觉系统重构（已完成）
- 由深色玻璃风改为高可读浅色体系，提升长时间写作可读性。
- 更新主色与语义色（主操作/警告/错误）对比关系，减少“颜色噪声”。
- 统一卡片、输入框、按钮、表格、芯片的圆角、阴影、边框语言。
- 统一标题/副标题/指标文本层级，减少页面跳读成本。
- 统一移动端样式断点，保证小屏可操作性。

### 2.2 交互体验重构（已完成）
- 新增移动端顶部快捷导航，解决“侧栏隐藏后无导航”的断层。
- 项目列表新增“关键词 + 状态”筛选条，支持快速定位项目。
- 新建项目弹窗支持：
  - 自动焦点
  - `Esc` 关闭
  - `Ctrl/Cmd + Enter` 提交
- 项目详情“整书导出”从占位升级为真实导出流程。
- 导出流程增加空正文保护，避免导出无意义文件。

## 3. 当前 UX 痛点地图（尽可能完整）

### A. 导航与信息架构
1. 移动端已补快捷导航，但桌面端仍缺“最近访问”与“最近章节”入口。
2. 项目详情、章节工作台、创作控制台之间缺“任务上下文跳转”（例如从冲突直接跳相关章节位置）。
3. 全局信息架构仍偏“页面分割”，缺任务导向路径（例如“从想法到成稿”的线性引导）。

### B. 核心创作流程
4. 批量生成缺清晰“进行中/可中断/可恢复”状态机展示。
5. 章节编辑缺自动保存和冲突解决向导，用户手动保存心智负担高。
6. 审批流提示可更强（为什么不能通过、下一步建议是什么）。
7. 一些按钮禁用时，缺“禁用原因”解释。

### C. 反馈与可观测性
8. Toast 反馈偏瞬时，缺“持久化操作历史”可回看。
9. 错误提示多数为通用文案，缺请求上下文与恢复建议。
10. 异步任务失败后重试入口不统一（有的页面有，有的没有）。

### D. 表单与输入体验
11. 表单校验主要靠按钮禁用，字段级错误提示不足。
12. 数值输入（章节数/字数）虽然有边界，但缺范围提示与推荐值解释。
13. 模态框在复杂场景下仍可增加“关闭前变更确认”。

### E. 视觉可读性与一致性
14. 多页面仍有大量内联样式，导致视觉规范难长期保持一致。
15. 图表区域与主内容区视觉权重仍可再平衡（看板信息密度偏高）。
16. 表格列信息在窄屏下仍有压缩风险（需更细粒度响应式策略）。

### F. 性能体验
17. 前端包体偏大，首屏感知速度与交互首次响应仍有优化空间。
18. 页面间重复请求可进一步缓存，减少来回切页等待。

### G. 可访问性
19. 键盘可用性已改善，但全站焦点态视觉一致性仍需加强。
20. 屏幕阅读器语义（状态变化、区域描述）还可以更完整。

## 4. 建议的 UX 改造路线（建议 3 个迭代）

### 迭代一：把“能用”变“顺手”
1. 补齐禁用原因提示与字段级校验。
2. 增加操作历史面板（最近生成、最近失败、最近导出）。
3. 统一错误恢复动作（重试、回退、查看详情）。

### 迭代二：把“顺手”变“高效”
1. 引入“任务向导”入口（从一句话到章节批量生产）。
2. 增加最近访问项目/章节快捷入口。
3. 自动保存草稿 + 冲突定位提示。

### 迭代三：把“高效”变“专业”
1. 路由级代码分割与缓存策略，优化首屏与切页体验。
2. 建立统一 Design Token + 组件规范，逐步替换内联样式。
3. 引入可访问性基线检查（键盘、对比度、语义标签）。
